# Frontend

## Principios

- Frontend e sistema: cada componente afeta performance, manutencao, acessibilidade e experiencia.
- Mobile-first e obrigatorio.
- Acessibilidade nao e opcional.
- Estados de loading, erro e vazio devem ser explicitos.
- Design system existente tem prioridade sobre opiniao local.

## React/Vite

- Prefira componentes pequenos com fronteiras claras.
- Evite memoizacao manual sem evidencia de necessidade.
- Nao introduza bibliotecas de estado sem justificativa concreta.
- Otimize bundle e waterfall de dados antes de micro-otimizacoes.

## UI

- Evite layouts genericos e intercambiaveis.
- Preserve tokens, escala tipografica e linguagem visual do projeto.
- Se nao houver design system, pergunte antes de criar UI nova.
